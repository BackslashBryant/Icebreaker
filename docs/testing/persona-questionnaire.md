# Persona Feedback Questionnaire

This questionnaire is designed to be answered by each persona during/after their testing journey. Each persona should answer truthfully based on their characteristics, concerns, and use case. The questions are designed to uncover UX friction points, feature gaps, and polish opportunities.

## Questionnaire Structure

Each persona answers:
1. **Onboarding Experience** (5 questions)
2. **Radar Discovery** (5 questions)
3. **Chat Interaction** (5 questions)
4. **Overall Experience** (5 questions)
5. **Persona-Specific Concerns** (3-5 questions tailored to persona)

**Scoring**: Each question uses a 1-5 scale (1 = Strongly Disagree, 5 = Strongly Agree) plus open-ended feedback.

## Feedback Collection Process

### During Testing
1. **Real-time notes**: Testers note friction points as they occur
2. **Screenshots**: Capture confusing or problematic UI moments
3. **Timing**: Note when tasks take longer than expected
4. **Emotions**: Note when personas feel anxious, frustrated, or relieved

### After Testing
1. **Questionnaire completion**: Each persona answers their specific questionnaire
2. **Open-ended feedback**: Encourage detailed written feedback beyond scales
3. **Prioritization**: Identify top 3 friction points per persona
4. **Pattern recognition**: Look for common issues across personas

### Feedback Analysis
1. **Quantitative**: Average scores per question, identify low-scoring areas
2. **Qualitative**: Thematic analysis of open-ended feedback
3. **Persona-specific**: Identify issues unique to persona types
4. **Prioritization**: Rank issues by severity and persona impact

---

## Usage Instructions

1. **Before testing**: Review persona characteristics and concerns
2. **During testing**: Answer questions as the persona would (not as yourself)
3. **After testing**: Complete questionnaire immediately while experience is fresh
4. **Be honest**: Answer truthfully based on persona's perspective, not ideal outcomes
5. **Be specific**: Provide concrete examples and suggestions in follow-up questions
